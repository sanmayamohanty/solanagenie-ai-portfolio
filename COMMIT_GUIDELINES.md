## üìù Commit Message Format

### Standard Format Template
```

- <change 1>

```

### Commit Types
- **feat**: New feature implementation
- **fix**: Bug fix or issue resolution
- **refactor**: Code restructuring without functional changes
- **style**: UI/UX improvements and styling changes
- **test**: Testing implementation or updates
- **docs**: Documentation updates (rare - avoid creating docs)
- **perf**: Performance improvements
- **build**: Build system or dependency changes
- **chore**: Maintenance tasks and configuration

## üéØ Commit Message Best Practices

### Descriptive Guidelines
1. **Be Specific**: Explain what was actually changed
2. **Use Action Words**: Start with active verbs
3. **Include Context**: Why the change was made if not obvious
4. **List Changes**: Use bullet points for multiple changes
5. **Professional Tone**: Write as if for a code review

### Length Guidelines
- **Subject Line**: 50-72 characters max
- **Description**: Detailed explanation with bullet points
- **Total Length**: No arbitrary limit - be thorough

### Technical Details
- **Function Names**: Include specific function/component names
- **File Paths**: Mention key files when relevant  
- **Dependencies**: Note new packages or version updates
- **Breaking Changes**: Clearly mark any breaking changes

## üö® Emergency Commit Procedures

### Critical Bug Fixes
```bash
# Create hotfix branch from main
git checkout main
git pull origin main
git checkout -b hotfix/critical-authentication-fix

# Make fixes and test thoroughly
# Commit with clear description
git commit -m "fix: resolve critical authentication token expiry

- implement proper token refresh mechanism
- add error handling for expired sessions
- prevent infinite redirect loops
- add user-friendly session timeout messaging"

# Push and create immediate PR
git push -u origin hotfix/critical-authentication-fix
```

### Rollback Procedures
```bash
# Rollback last commit (if not pushed)
git reset HEAD~1

# Revert specific commit (if pushed)
git revert <commit-hash>

# Emergency rollback to specific commit
git reset --hard <commit-hash>
git push --force-with-lease
```
